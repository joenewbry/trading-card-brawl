# Contributing to Trading Card Brawl

## Branch Naming

Use the following prefixes for branch names:

- `feature/` — New functionality
- `fix/` — Bug fixes
- `chore/` — Maintenance, tooling, dependencies

Example: `feature/add-new-card-type`

## Commit Messages

Use the conventional commits format:

- `feat:` — New feature
- `fix:` — Bug fix
- `chore:` — Build process or tooling change
- `docs:` — Documentation only

Example: `feat: add new fire card type`

## Pull Request Workflow

1. Link your PR to a task in `TASK_TRACKER.csv` (include the ID in the PR description)
2. Fill out the PR template below
3. Ensure all CI checks pass
4. Request at least one reviewer before merging
5. Do not merge your own PR — wait for approval

## Branch Protection Rules

The `main` branch is protected:
- Require pull request reviews before merging
- Require status checks to pass
- Include administrators

Do not bypass these rules.
